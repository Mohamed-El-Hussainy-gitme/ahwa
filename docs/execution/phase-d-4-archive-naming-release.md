# phase d / batch 4 - archive discipline, naming review, and release checks

هذه الدفعة تنظف الطبقة الإدارية والوثائقية فوق النسخة الناتجة من الدفعة 3، بدون توسيع الدومين.

## scope

- **archive discipline**
  - القوائم اليومية تبدأ بالعناصر النشطة فقط
  - العناصر المؤرشفة أو المعطلة لا تظهر افتراضيًا إلا عند المراجعة
  - ينطبق هذا على القهاوي في لوحة السوبر أدمن، وعلى المنيو، وعلى الموظفين
- **naming review**
  - توحيد المصطلحات الأساسية بين الـ UI والوثائق
  - إزالة labels الإنجليزية من tabs السوبر أدمن الأساسية
  - تثبيت term mapping نهائي مختصر
- **release checks**
  - إضافة فحص آلي يتأكد أن discipline هذه ما زالت موجودة ولم تُكسر بتعديل لاحق

## code changes

### super admin dashboard
- tabs الرئيسية أصبحت:
  - `النظرة العامة`
  - `القهاوي`
  - `المتابعة المالية`
- فلتر القهاوي يبدأ افتراضيًا على `المفعلة فقط`
- هذا يجعل الشاشة أسرع وأكثر عملية عند وجود عدد كبير من المقاهي

### menu
- المنيو اليومي يعرض الأقسام والأصناف النشطة فقط افتراضيًا
- يوجد toggle واضح لإظهار `المؤرشف/المعطل` عند الحاجة للمراجعة فقط

### staff
- صفحة الموظفين تستمر في البدء بالنشطين فقط
- تم توضيح هذا في الواجهة والنصوص التنظيمية

## naming review output
- `docs/term-mapping.md` أصبح المرجع المختصر للتسمية canonical
- تمت إضافة mapping واضح لـ:
  - item issue
  - deferred ledger
  - left / inactive employee
  - super admin labels

## release checks
- script جديد:
  - `scripts/check-batch4-admin-hygiene.mjs`
- يتم استدعاؤه من release readiness للتأكد من:
  - labels العربية في السوبر أدمن
  - discipline الافتراضية للقوائم
  - وجود docs الخاصة بهذه الدفعة

## outcome
- الواجهات اليومية تبقى أنظف
- السوبر أدمن أصبح أوضح وأكثر اتساقًا
- naming drift أقل
- release checks تغطي هذه الدفعة حتى لا تضيع في تعديلات لاحقة
