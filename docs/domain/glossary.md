# Domain glossary

## Core terms

- **cafe**: tenant boundary for isolation and operations
- **shift**: operational frame for all daily work
- **service session**: temporary label-based runtime container opened by the first order
- **order**: command envelope for submitted items
- **order item**: quantity-tracked operational unit
- **station**: execution lane (`barista`, `shisha`, `service`)
- **deferred ledger**: customer-name based debt / repayment ledger
- **shift snapshot**: persisted end-of-shift summary

## Explicitly retired terms

These terms are no longer part of the canonical runtime vocabulary:

- table
- table session
- bill account
- deferred account as a separate master entity in daily runtime
